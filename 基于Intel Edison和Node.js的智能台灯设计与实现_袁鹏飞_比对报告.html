摘 要

光是生活中无处不在的重要元素，灯则集成了这种分布广，使用频率高，需求性强的特性，照明系统可以说是家中最重要，分布最广，使用频率最高，而且是日常生活中的刚性需求。灯具作为中最为普及的电器，从发明之初，就进入了家家户户的日常生活当中。因此，作为家庭的必需用品，智能灯具也可以作为智能家居的一个重要入口，培养使用人群，提升人们的生活质量。

本次设计实现的是基于Intel Edison和Node.js开发平台的智能台灯，以综合分析目前智能家居设备中终端设备的开发背景及市场,并且同时探究智能台灯作为智能化家居设备在智能家居系统中的发展方向。在设计的过程中主要使用的开发语言为HTML、CSS、JavaScript。在整个设计过程中着重的的学习了Express.js和Socket.io两个Node.js模块的使用，并且了解了MRAA库控制Intel Edison的GPIO的方法，同时还加深了对前端框架Material Design Lite的理解。通过学习和尝试，本次毕业设计最终成功的实现了智能台灯预期的功能并进行了软硬件的集成测试，最终的测试结果达到设计目标。

关键词：智能家居；智能台灯；Node.js；Intel Edison.

第一章 绪论

1.1 课题研究背景

在信息技术的快速发展的时代，人们对智能家居改变生活有了更多的期盼，随着生活水平的提高和中产阶级的不断壮大，在物联网产品井喷式出现的现如今，满足提升生活品质追求变得日益迫切。早晨起床时，新鲜的水煮蛋已经做好；夜里起床时，智能夜灯会自动亮起，智能家居就这样从身边的小事做起，逐渐走进了人们的生活。

智能家居是家居领域的必然趋势，而且产业前景诱人，这就毫无悬念的引发了从国内外诸多行业巨头进入智能家居领域抢占市场[2]，国内走在前线的有小米公司的“小米智能家居系统”，海尔的“U-Home”，国外做的比较出色的有被Google以32亿美元收购的“Nest”智能家庭系统，Amazon的“Alexa”则以一个智能音响开始，逐步构建出一个智能家居的生态链。

现在市场上大量销售的台灯，绝大部分仍是采用普通的220v交流电的白炽灯、荧光灯，功耗较大[7]，控制方式多采用手动开关，很少有能进行亮度调节的，更不能进行冷暖色调节。天黑时，人们又摸黑去开灯，十分的不方便，与现在家电的智能化，人性化，低碳设计理念相违背[3]。

1.2 课题研究意义

随着各种智能家具的开发，我们日常中重要的台灯也应该顺应科技的发展步伐走向智能化。

近年来中国灯具行业发展迅速，受益于灯具行业生产技术不断完善，下游市场需求持续扩大，灯具行业在国内外市场发展状况十分乐观。虽然近年来受到金融危机的音响，灯具行业发展速度趋于缓慢，但随着国民经济的快速发展和国际金融危机的衰退，中国灯具行业将重新迎来发展机遇。

进入2010年以来，我国台灯行业面临全新的发展趋势，由于新兴产业的不断增多，上游原材料的价格持续暴涨，导致行业的利润下降，因此我国台灯行业的市场竞争也日趋激烈。面对如此的形式，提升企业的创新能力，提高产品的技术特点，把握家具市场智能化的态势，运用最新的技术到自己的产品当中，才能使企业在残酷的市场竞争中取得领先的优势。

Intel Edison 的出现满足了广大电子爱好者、创客和开发人员的需求，它性能强劲、可拓展性强、又简单易用、功耗低、体积小巧放置进台灯也没有问题。而Node.js优秀的拓展性和极低的内存占用又有事件驱动，非阻塞I/O模型，执行效率也十分高效使得Node.js也十分适合各种嵌入式设备上运行，多种优秀的特性使得Node.js适应了智能硬件开发的潮流，也得到了Intel的青睐，为其开发了专门的IDE以及官方的支持库。

本系统使用了先进的智能硬件开发平台Intel Edison，运用了Node.js作为主要的开发语言，前端使用Google的Material Design设计风格，实现智能台灯的相应功能，界面简洁，操作方便，系统设计了亮度调节、冷暖色温调节、专注模式、休息提醒、模拟日光唤醒、成就系统等众多功能，更大程度的让用户体验到最人性化的智能而过台灯体验。系统完成度较高、功能明确、界面美观、富有人性化，实现了最初的设计目的，足以成为智能台灯行业的一项重要试水。

台灯产品很多，一款台灯有两个基本需求，第一就是要一定的遮光性——当人处于正常的坐姿情况下，眼睛向前看，应该看不到灯罩的内壁和光源，第二就是照度均匀的要求——应该确保受台灯照射的工作区域内，照度相对均匀，不会产生特别亮或者暗的光斑[6]，只有首先满足这两点，才是一款合格的台灯。

以往的大多数台灯使用的都是普通的白炽灯，但普通白炽灯本身存在着寿命短，能耗高等方面的缺点，因此，进入新的时代，白炽灯已不再适合作为家居照明的首选光源，现代科学技术在快速发展，LED是当前社会比较热门的光源技术之一，它利用特殊材料制成的PN结在正向电压的作用下，点子与空穴复合产生光子，从而实现发光，所以它是直接将电能转化为光能，而白炽灯在工作中有95%的能量都转化为乐内能散发到空气中，所以相比较与白炽灯，LED光源的发热要小很多而且寿命更长，与节能灯相比，它具有更高的电光转换效率，而且不含对人体有害的重金属元素，同时制造成本十分低廉，绝对是新时代的优质光源。

所以在整个智能台灯光源的选择上决定使用双色温贴片LED，其光效高、工作电压低、能耗低、无辐射、同样亮度下的能耗为白炽灯的10%、荧光灯的50%，并且整个可以实现色温的无极调整。智能台灯选择LED光源更加符合低碳的设计理念。通过并联多个小型LED灯板，最终不需要施加太大的电压，就可以达到足够的照明亮度。

3.2 JavaScript和Node.js介绍

3.2.1 JavaScript是什么

JavaScript是一种轻量级的脚本语言。所谓“脚本语言”，指的是它不具备开发操作系统的能力，而是只用来编写控制其他大型应用程序的“脚本”。[10]

JavaScript是一种嵌入式（embedded）语言。它本身提供的核心语法，规模相当小，只能用来做一些数学和逻辑运算。JavaScript本身不提供任何与I/O（输入/输出）相关的API，都要靠宿主环境（host）提供，所以JavaScript只合适嵌入更大型的应用程序环境，去调用宿主环境提供的底层API[10]。

目前，已经嵌入JavaScript的宿主环境有多种，最常见的环境就是浏览器，另外还有服务器环境，也就是Node项目。

从语法角度看，JavaScript语言是一种“对象模型”语言。各种宿主环境通过这个模型，描述自己的功能和操作接口，从而通过JavaScript控制这些功能。但是，JavaScript并不是纯粹的“面向对象语言”，还支持函数式编程[10]。

JavaScript的核心语法部分相当精简，只包括两个部分：基本的语法构造（比如操作符、控制结构、语句）和标准库（就是一系列具有各种功能的对象比如Array、Date、Math等）。除此之外，各种宿主环境提供额外的API（即只能在该环境使用的接口），以便JavaScript调用。以浏览器为例，它提供的额外API可以分成三大类。

 浏览器控制类：操作浏览器

 DOM类：操作网页的各种元素

 Web类：实现互联网的各种功能

JavaScript的发明目的，就是作为浏览器的内置脚本语言，为网页开发者提供操控浏览器的能力。它是目前唯一一种通用的浏览器脚本语言，所有浏览器都支持。它可以让网页呈现各种特殊效果，为用户提供良好的互动体验[1]。

目前，全世界几乎所有网页都使用JavaScript。如果不用，网站的易用性和使用效率将大打折扣，无法成为操作便利、对用户友好的网站。

对于一个互联网开发者来说，如果你想提供漂亮的网页、令用户满意的上网体验、各种基于浏览器的便捷功能、前后端之间紧密高效的联系，JavaScript是必不可少的工具。近年来，JavaScript的使用范围，慢慢超越了浏览器，正在向通用的系统语言发展。

3.2.2 Node.js是什么

Node.js，或者 Node，是一个可以让 JavaScript 运行在服务器端的平台。它可以让JavaScript 脱离浏览器的束缚运行在一般的服务器环境下，就像运行 Python、Perl、PHP、Ruby程序一样[13]。你可以用 Node.js 轻松地进行服务器端应用开发，Python、Perl、PHP、Ruby 能做的事情 Node.js 几乎都能做，而且可以做得更好。

Node.js 是一个为实时 Web（Real-time Web）应用开发而诞生的平台，它从诞生之初就充分考虑了在实时响应、超大规模数据要求下架构的可扩展性。这使得它摒弃了传统平台依靠多线程来实现高并发的设计思路，而采用了单线程、异步式I/O、事件驱动式的程序设计模型。这些特性不仅带来了巨大的性能提升，还减少了多线程程序设计的复杂性，进而提高了开发效率[11]。

Node.js 最初是由 Ryan Dahl 发起的开源项目，后来被 Joyent 公司注意到。Joyent 公司将Ryan Dahl 招入旗下，因此现在的 Node.js 由 Joyent 公司管理并维护。尽管它诞生的时间（2009年）还不长，但它的周围已经形成了一个庞大的生态系统。Node.js 有着强大而灵活的 包管理器 （node package manager，npm），目前已经有上万个第三方模块，其中有网站开发框架，有 MySQL、PostgreSQL、MongoDB 数据库接口，有模板语言解析、CSS 生成工具、邮件、加密、图形、调试支持，甚至还有图形用户界面和操作系统 API工具[12]。

Node.js 不是一种独立的语言，与 PHP、Python、Perl、Ruby 的“既是语言也是平台”不同。Node.js 也不是一个 JavaScript 框架，不同于 CakePHP、Django、Rails。Node.js 更不是浏览器端的库，不能与 jQuery、ExtJS 相提并论。Node.js 是一个让 JavaScript 运行在服务端的开发平台，它让 JavaScript 成为脚本语言世界的一等公民，在服务端堪与 PHP、Python、Perl、Ruby 平起平坐。

Node.js 是一个划时代的技术，它在原有的 Web 前端和后端技术的基础上总结并提炼出了许多新的概念和方法，堪称是十多年来 Web 开发经验的集大成者[13]。Node.js 可以作为服务器向用户提供服务，与 PHP、Python、Ruby on Rails 相比，它跳过了 Apache、Nginx 等 HTTP服务器，直接面向前端开发。Node.js 的许多设计理念与经典架构（如 LAMP）有着很大的不同，可提供强大的伸缩能力，以适应21世纪10年代以后规模越来越庞大的互联网环境，重要的是Node.js使用了非阻塞I/O模型而得以在相对较低的系统资源消耗下表现出高性能和优秀的负载能力，这样的一些特性使得Node.js十分适合工作在物联网设备上。

3.2.3 Node.js与JavaScript

说起 JavaScript，不得不让人想到浏览器。传统意义上，JavaScript 是由 ECMAScript、文档对象模型（DOM）和浏览器对象模型（BOM）组成的，而 Mozilla 则指出 JavaScript 由Core JavaScript 和 Client JavaScript 组成。之所以会有这种分歧，是因为 JavaScript 和浏览器之间复杂的历史渊源，以及其命途多舛的发展历程所共同造成的。我们可以认为，Node.js 中所谓的 JavaScript 只是 Core JavaScript，或者说是 ECMAScript 的一个实现，不包含 DOM、BOM 或者 Client JavaScript。这是因为 Node.js 不运行在浏览器中，所以不需要使用浏览器中的许多特性。

Node.js 是一个让 JavaScript 运行在浏览器之外的平台。它实现了诸如文件系统、模块、包、操作系统 API、网络通信等 Core JavaScript 没有或者不完善的功能。历史上将 JavaScript移植到浏览器外的计划不止一个，但Node.js 是最出色的一个。随着 Node.js 的成功，各种浏览器外的 JavaScript 实现逐步兴起，因此产生了 CommonJS 规范。CommonJS 试图拟定一套完整的 JavaScript 规范，以弥补普通应用程序所需的 API，譬如文件系统访问、命令行、模块管理、函数库集成等功能。CommonJS 制定者希望众多服务端 JavaScript 实现遵循CommonJS 规范，以便相互兼容和代码复用。Node.js 的部份实现遵循了CommonJS规范，但由于两者还都处于诞生之初的快速变化期，也会有不一致的地方[15]。

Node.js 的 JavaScript 引擎是 V8，来自 Google Chrome 项目。V8 号称是目前世界上最快的 JavaScript 引擎，经历了数次引擎革命，它的 JIT（Just-in-time Compilation，即时编译）执行速度已经快到了接近本地代码的执行速度。Node.js 不运行在浏览器中，所以也就不存在 JavaScript 的浏览器兼容性问题，所以可以放心地使用 JavaScript 语言的所有特性。

为了完成页面控件实时滑动就能改变台灯亮度和冷暖色调的调整的目的，这就要使用到Web领域的实时推送技术，也被称作Realtime技术。这种技术要达到的目的是让用户不需要刷新浏览器就可以获得实时更新。它有着广泛的应用场景，比如在线聊天室、在线客服系统、评论系统、WebIM等。

一、WebSocket简介

谈到Web实时推送，就不得不说WebSocket。在WebSocket出现之前，很多网站为了实现实时推送技术，通常采用的方案是轮询(Polling)和Comet技术，Comet又可细分为两种实现方式，一种是长轮询机制，一种称为流技术，这两种方式实际上是对轮询技术的改进，这些方案带来很明显的缺点，需要由浏览器对服务器发出HTTP request，大量消耗服务器带宽和资源。面对这种状况，HTML5定义了WebSocket协议，能更好的节省服务器资源和带宽并实现真正意义上的实时推送。

WebSocket协议本质上是一个基于TCP的协议，它由通信协议和编程API组成，WebSocket能够在浏览器和服务器之间建立双向连接，以基于事件的方式，赋予浏览器实时通信能力。既然是双向通信，就意味着服务器端和客户端可以同时发送并响应请求，而不再像HTTP的请求和响应。

为了建立一个Web Socket连接，客户端浏览器首先要向服务器发起一个HTTP请求，这个请求和通常的HTTP请求不同，包含了一些附加头信息，其中附加头信息“Upgrade: WebSocket”表明这是一个申请协议升级的HTTP请求[16]，服务器端解析这些附加的头信息然后产生应答信息返回给客户端，客户端和服务器端的WebSocket连接就建立起来了，双方就可以通过这个连接通道自由的传递信息，并且这个连接会持续存在直到客户端或者服务器端的某一方主动的关闭连接。

3.3.3 MRAA

MRAA 是用 Intel开发的一个用C 语言编写的低级别通用库，同时它提供了一个Node.js版本的类库。 MRAA 旨在提取GPIO的功能访问和控制相关的详细信息，并将其转化为单个、简洁的 API。

MRAA 可作为 Linux 通用输入/输出 (GPIO) 设备顶端的转换层。 尽管 Linux 提供丰富的方法来控制 GPIO，其用于处理 GPIO 的通用指令也非常标准，但使用难度相对较大，这也正是我们放弃使用C\C++开发智能台灯的原因。

Intel Edison的官方其实一共有两块拓展板子，除了次使用的Arduino拓展板之外，还有一块Breakout拓展板，由于设计中不会使用，所以这里也不在多说。官方的拓展板之间各具差异， 他们拥有不同的功能、针脚编号和 GPIO 类型。 例如，更换拓展板后，GPIO 针脚可能无法支持相同类型的功能。甚至原始的Intel Edison本身甚至可能根本没有针脚。 另外， GPIO 的配置方式也取决于不同因素。 例如，一种 GPIO 针脚使用模式可能会影响其他针脚的其他使用模式，或影响其他针脚的使用。 因此，Intel使用了MRAA 来降低程序开发的复杂度，因为它可以识别开发平台的类型，始终保证程序对GPIO操作是正确的。

4.1 硬件电路连接

智能台灯选用的是双色温LED光源，在灯板上有两种色温的LED灯珠，分别是白色和黄色的，通过调节两种灯珠的亮度来实现色温的调节和整体照明亮度的调节，要实现LED灯珠的亮度调节，主要参考的方法有两种：

一、 线性调光

因为LED的亮度是几乎和它的驱动电流直接成正比关系，所以可以通过改变它的驱动电流来实现。调节LED的亮度最简单的方法就是由芯片提供一个控制电压接口，改变输入的控制电压就可以改变其输出恒流值。

但此种方法会造成在调亮度地同时也会改变它的光谱和色温，因为目前白光LED都是用蓝光LED激发黄色荧光粉而产生，当正向电流减小时，蓝光LED亮度增加而黄色荧光粉的厚度并没有按比例减薄，从而使其光谱的主波长增长，改变色温；[17]而且从LED的伏安特性可知，正向电流的变化会引起正向电压的相应变化，确切地说，正向电流的减小也会引起正向电压的减小。所以在把电流调低的时候，LED的正向电压也就跟着降低。这就会改变电源电压和负载电压之间的关系，引起恒流源无法工作的严重问题[18]。

二、 PWM调光

LED是一个二极管，它可以实现快速开关。它的开关速度可以高达微秒以上，是任何发光器件所无法比拟的。因此，只要把电源改成脉冲恒流源，用改变脉冲宽度(PWM)的方法，就可以改变其亮度[18]。

脉宽调制调光的优点不会产生任何色谱偏移可以有极高的调光精确度；可以和数字控制技术相结合来进行控制即使在很大范围内调光，也不会发生闪烁现象[20]。

所以选择使用宽脉调制调光的工作方式用来改变光源的亮度。

因为Intel Edison的PWM输出功率不足以支持选用的光源正常工作，所以需要额外的LED光源驱动电路。在本次的设计中选用了L289N双H桥电机驱动模块，使用它可以成功驱动光源，它本身还支持PWM的输入。

就目前而言，本系统不足的地方还有不少，比如连接的方式还比较复杂。第一次使用，需要用户具有一些Linux操作的经验，才能使设备成功接入的家庭的网关，而后才能进行控制的操作。假如改成使用蓝牙进行连接的方式，然后设备自动获取到手机连接到的家庭网关，从而能够自动连接并且返回给用户控制界面的地址，这样就会比现在更加实用。其次，编写的代码在工作中使用的是不断轮询的方式，虽然Intel Edison性能足够强大能够应付这样的操作，但长期如此运行，对系统的负担还是很重，也会影响设备的使用寿命，假如可以改用事件触发机制，就会减少大量的系统占用，使得系统运行更加稳定。

5.2 特色展望

当前设计的智能台灯仅仅使用了一小部分Intel Edison的性能，但可以展望的还很多，比如现在系统中运行着的专注模式，进一步升级后还可以收集到的用户习惯，上传到云端进行分析，帮助用户了解自己在什么时候，有着更高的工作效率，为用户的生活和工作，提供相关的建议。

又或者，在设备上可以添加许多其他模块，超声波测距模块，用来提醒用户阅读或办公坐姿是否正确；USB充电模块，为设备增加几个USB充电口，并且对充电状态进行实时检测，当电充满时提醒用户拔下充电设备，不仅防止过冲，还为节能环保贡献一份力量；睡眠检测模块，通过监测用户睡眠状态，提出健康建议，从而帮助用户了解自己的睡眠习惯，睡眠状态；将Intel Edison打造成一个家庭控制系统，比如配合IOS的Homekit控制系统，控制家庭内的其他用电器，让它们按照那个用户的需求远程工作……

可以展望的还有太多太多，基于Intel Edison和Node.js的智能台灯仅仅是一次使用Node.js来进行智能家居开发的小小的尝试，但开创性的使用了Intel Edison来进行智能家居方面的研究，相信随着人们需求的不断提升，智能家居必将成为人们生活中不可缺少的重要部分。

结束语

现如今，智能家居正在借着物联网浪潮之势力，来到人们的生活当中，逐步满足人们期盼拥有的安全、舒适节能的家居环境，随着智能设备成本降低和功能的提升，这股家居智能化的力量更是势不可挡。

本次毕业设计开发的基于Intel Edison和Node.js的智能台灯系统，主要包括了软件和硬件两个方面。在开发的过程中，根据硬件的特点，选择了合适的开发平台，开发工具，根据设计的需求，从用户的角度进行开发，用模块化设计方法将系统的各个功能模块层层分解，设计出系统的模块化结构，最终实现了智能台灯的设计。

本系统在前端界面上是用Google的Material Design，虽然设计整体的饱和度不低，但却并不杂乱。使用了Material Design Lite进行了响应式开发，可以完美适应不同分辨率的设备。利用Intel Edison强大的性能和对GPIO的操作进行硬件，得益于Intel XDK的方便使用，使得开发调试变得十分方便。

在开发的过程中，也遇到了一些困难，例如：需要需要的功能无法实现，需要对框架进行大量的修改；需要设计的功能始终无法实现等。但在咨询老师、查阅资料，多次修改和测试后解决了这些问题。最后，系统成功的进行了功能测试，功能运行正常，并且达到了预期的效果，不仅如此，一些重要功能上还有着独特的创新[22]。

但是由于一些时间和能力的因素，系统的功能还比较少，Intel Edison强大的性能还未能进一步释放，还有很多功能可以继续进行开发和优化。Node.js也仅仅只用到了几个常用的模块，更多强大的功能还未能掌握。在以后的工作学习中，还将带着以上的一些经验和欠缺继续思考。
